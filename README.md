## Общие сведения

Данное приложение позволяет создавать большие объёмы синтетических данных для тестирования, аналитики, демонстраций и разработки.

---

## Интерфейс приложения

Основной интерфейс разделён на несколько секций:

1. **Настройки генерации** — имя файла, количество строк, формат, язык, тип генерации.
2. **Загружаемые источники данных** — загрузка внешних файлов (CSV/JSON/XML/QVD/XLSX).
3. **Столбцы** — конфигурация каждого столбца и данных будущего файла.
4. **Предпросмотр** — отображение первых 10 строк результата.
5. **Кнопки действий** — генерация, сохранение/загрузка настроек, выбор темы.

---

## Типы генерации

Приложение поддерживает два режима:

**Упрощённая (Быстрая)**
Использует сокращённые наборы имён, регионов и городов. Подходит для быстрой генерации без детализации географических/персональных данных.

**Расширенная (Медленная)**
Использует полные наборы данных. Требует больше памяти и времени генерации, но обеспечиваетрасширенную вариативность персональных/географических данных.

Выбор влияет на:
- объём имён,
- список стран и регионов: в упрощённой генерации для геоданных участвуют **только США, Россия и Испания**.
- количество городов и улиц.

Для генерации данных так же **используется распределение Парето**, чтобы увеличить количество выбросов при псевдорандоме.

## Форматы выходных файлов

| Формат | Расширение | Особенности |
|--------|------------|-------------|
| CSV | `.csv` | Текстовый формат с изменяемым разделителем, поддерживает экранирование кавычек. |
| XLSX | `.xlsx` | Excel-файл, удобен для анализа в таблицах. |
| XML | `.xml` | Структура: `<rows><row><col1>...</col1>...</row></rows>`. |
| JSON | `.json` | Массив объектов, каждый объект - строка данных. |
| QVD | `.qvd` | Формат QlikView/Qlik Sense. |

> ⚠️ Для XML файлов происходит замена символов под стандарт написания xml тегов, названия столбцов выходного файла могут отличаться от прописанных в настройках генерации.
> ⚠️ Для QVD файлов установлено ограничение в 50000 строк из-за ограничения формата.

### Текстовые данные

| Тип | Параметры | Описание |
|-----|----------|---------|
| **Строка случайной длины** | Базовый текст, макс. длина | Возвращает подстроку от начала до случайной длины (0–max). |
| **Кастомный список** | Список значений, режим выбора, разделитель | Кастомный список служит для генерации данных из указанных значений. Поддерживает режимы: случайный, последовательный, последовательный (игнорируя размер). Для последнего варианта включается игнорирование глобальной настройки количества строк и определяется количеством элементов списка (удобно, когда необходимо создать связи разных наборов 1:1). |
| **Преобразования** | Формула (JavaScript) | Вычисляет значение на основе других столбцов текущей строки. Формула выполняется после генерации всех значений в строке. |
| **Всегда пустая строка** | — | Для всех значений столобца возвращает пустые строки `""`. |
| **Всегда null** | — | Для всех значений столобца возвращает `null`. |
| **Капризные строки** | — | Случайные строки с различными символами разных кодировок, спецсимволами и эмодзи для проверок обработки/экранирования данных. |

#### Преобразования (JavaScript-формулы)

Тип **Преобразования** позволяет рассчитать значение столбца через JavaScript-выражение.

Доступ к данным текущей строки:

- `row["Имя столбца"]` — прямой доступ к значению столбца.
- `col("Имя столбца")` — безопасный хелпер (если столбец не найден, вернёт пустую строку).

Правила написания формул:

- Формула должна быть **одним JavaScript-выражением** (без `return` и без многострочного кода).
- Можно использовать стандартные функции JS, например: `Number(...)`, `String(...)`, `Math.round(...)`.
- Формула выполняется **после генерации всех столбцов** текущей строки, поэтому можно ссылаться на любые уже настроенные столбцы.
- При ошибке вычисления (например, синтаксическая ошибка) значение столбца будет пустой строкой.

Примеры:

- `row["цена"] * row["количество"]`
- `Number(col("сумма")) / 100`
- `String(col("имя")) + " " + String(col("фамилия"))`
- `(Number(col("доход")) - Number(col("расход"))) || 0`

### Географические данные

| Тип | Наследование | Ограничения |
|-----|--------------|------------|
| **Коды стран** | — | ISO 3166-1 alpha-2 (`RU`, `US`, `DE`). |
| **Страны** | От кода страны | Полное название на выбранном языке. |
| **Регионы** | От страны / кода страны | Можно ограничить список стран через мультиселект. |
| **Города** | От региона / страны / кода страны | Автоматически выбирает город из соответствующей страны/региона. |
| **Улицы** | От города / региона / страны / кода страны | Аналогично городам. Содержит усреднённый список самых повторяемых названий улиц для стран. |

| **Линии** | От города / региона / страны / кода страны | Список точек внутри полигона. Поддержка `PIX BI` и `GeoJSON`. |

Важно заметить, что при наличии поля с типом "Регион" генерация с наследованием будет в рамках региона. Если для региона нет данных, то генерация будет в рамках страны региона. Если нет данных для страны, то генерация координат будет случайной.
> Так же для координат при наличии поля с типом "Города" и включенным наследованием берётся точка в центре города и в радиусе 4км от центра генерируются случайные координаты.

### Данные из внешних файлов

Вы можете загрузить файлы (CSV/JSON/XML/QVD/XLSX), и использовать их столбцы как источники данных.

- Поддерживается выбор нескольких файлов.
- Поддерживается режим: **последовательно (игнорируя размер)** — генерирует столько строк, сколько строк в источнике.
- Если файл удалён из списка, но столбец остался — он помечается как «⚠️ Удалённый файл» и вместо данных для столбца добавляются пустые строки.
- При клике на имя файла в списке — все его столбцы добавляются автоматически. Полезно для того, чтобы пересобрать файл в другом формате.

Фактически, чтобы пересобрать файл из одного формата в другой необходимо загрузить файл, нажать на этот файл, выбрать необходимый формат, нажать "Сгенерировать файл".

> ✅ **Важно**: при добавлении столбцов через нажатие на файл режим выбора для каждого столбца автоматически устанавливается на "Последовательно (игнорируя размер)", что игнорирует глобальную настройку количества строк при генерации.

## Наследование значений (подробнее)

Некоторые типы данных могут **наследовать** значение от других столбцов:

| Наследуемый тип | От какого столбца может наследовать |
|------------------|-------------------------------------|
| Страна | Код страны |
| Регион | Страна / код страны |
| Город | Регион / Страна / Код страны |
| Улица | Город / Регион / Страна / Код страны |
| Имя/Фамилия/Отчество | Пол |
| Координаты | Город / Регион / Страна / Код страны |
| Линии | Регион / Страна / Код страны |
| Телефон |  Регион / Страна / Код страны |

> ✅ **Важно**: наследовать значения можно от любого поля с соответствующим, указанным выше, типом.

## Пропуски (null-значения)

Для любого столбца можно задать вероятность пропуска значений:

- Указывается в процентах (0–100).
- При генерации с заданной вероятностью значение заменяется на `null`.

---

## Импорт и экспорт настроек

В приложении не предусмотрено автоматическое сохранение настроек, но есть возможность сохранить текущую конфигурацию для последующего использования.

- **Сохранить настройки** → создаёт `.json`-файл с полной конфигурацией (столбцы, параметры, язык и т.д.).
- **Загрузить настройки** → восстанавливает интерфейс из файла.
- Имя файла настроек по умолчанию включает: базовое имя, количество строк, формат, язык, тип генерации и дату.

## Работа с внешними источниками данных

Поддерживаемые форматы:

| Формат | Особенности парсинга |
|--------|----------------------|
| **CSV** | Автоматическое определение колонок, обработка кавычек и переносов. Принимает csv только с базовым разделителем - запятой. |
| **JSON** | Принимает только массив объектов; заголовки — объединение всех ключей. |
| **XML** | Ожидается структура `<rows><row><col>...</col></row></rows>`. |
| **XLSX** | Обрабатываются все листы. Каждый лист становится отдельным источником. |
| **QVD** | Временный файл создаётся во время парсинга. |

## Предпросмотр данных

- Отображает **первые 10 строк**.
- Автоматически обновляется при изменении столбцов.
- Можно обновить вручную кнопкой **«Обновить предпросмотр»**.
- При ошибке генерации вместо данных показывается сообщение об ошибке. Большая часть ошибок обработана.

> ✅ **Полезное**: При нажатии на ячейку в таблице предпросмотра значение ячейки копируется в буфер обмена.

## Темы оформления

Поддерживается три режима:

- **Системная** — следует настройкам ОС.
- **Светлая** — светлое оформление интерфейса.
- **Тёмная** — тёмное оформление интерфейса.

Переключается кнопкой **🌓 Системная** в правом нижнем углу.

## Советы и рекомендации

1. **Для больших объёмов (>100 000 строк)** используйте **упрощённую генерацию** — она быстрее и потребляет меньше памяти.
2. **Гео-наследование** работает только если есть полигон страны/региона (а есть не для всех стран, подробнее в разделе [Географические данные](#географические-данные)).
3. При использовании **внешних файлов** убедитесь, что они не содержат пустых строк в начале. Сценарий частично обработан, но на всякий случай стоит.
4. В приложении присутствует undo/redo по **Ctrl+Z/Cmd+Z** для undo и **Ctrl+Y/Ctrl+Shift+Z** и **Cmd+Y/Cmd+Shift+Z** для redo. Но только на 40 изменений.
